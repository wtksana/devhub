# 当前状态与下一步

## 当前约定

- 后续直接在 `C:\Dev\devhub` 的 `main` 分支开发。
- 不再使用 `.worktrees` 目录作为日常开发位置。
- 文档、提交信息、对话总结默认使用简体中文。
- AI 相关功能已移出当前产品范围，后续不在 MVP 中继续推进。
- UI 设计继续参考 Zed 的工作台风格：紧凑、工具型、面板清晰、标签栏整体性强。
- 主题使用 Atom One Dark / One Light 的配色方向。
- 设置采用图形界面和 `settings.json` 双向编辑。
- 界面语言支持 `zh-CN` / `en-US`，设置项默认值为 `system`，系统中文使用简体中文，其他系统默认 English。
- 连接、外观、布局、快捷键等配置优先放入可迁移 JSON。
- SSH 密码按当前产品约定直接保存在连接配置中。
- Redis 密码按当前产品约定直接保存在连接配置中。
- 私钥内容不写入配置，只保存私钥路径。
- 私钥口令按当前产品约定直接保存在连接配置 `auth.passphrase` 中。
- 终端、SFTP、数据库、Redis 等长连接能力优先由 Rust 后端管理。
- SFTP 基础文件管理器属于第一版范围。

## 最近完成

- 移除了 AI 面板、AI 设置和后端 AI 相关代码。
- 连接面板支持添加连接弹窗。
- 连接面板按分组展示连接，未填写分组的连接显示在“未分组”中。
- 本地终端固定显示在连接面板顶部，不进入任何分组。
- 连接项右键菜单支持移动到分组，移动到“未分组”会清除连接分组。
- 连接面板列表空白处右键菜单支持添加连接、添加分组、分组排序和连接排序。
- 连接项支持双击打开终端。
- 连接项支持右键菜单：连接、新标签连接、SFTP、编辑、复制。
- 工作区标签支持右键菜单：关闭、关闭其他、关闭左侧、关闭右侧。
- 空白工作区支持右键菜单：打开设置、显示连接面板。
- 终端区域支持右键菜单：复制、粘贴、清屏。
- 终端剪贴板改用 Tauri 剪贴板插件，避免浏览器剪贴板权限弹窗。
- 终端粘贴、复制、清屏后会恢复焦点。
- SSH 读取中的 `transport read` 瞬时错误不会直接显示到终端里。
- 终端打开时会先注册输出监听，再创建后端会话；远端输出早于 `open_terminal` 返回时会临时缓存，避免偶发卡在 `Connected.`。
- 已切回主目录 `main` 分支开发，并删除 `.worktrees` 目录。
- 应用关闭时会保存窗口大小，下次启动优先恢复上次尺寸；默认尺寸仍以 1280x720 兜底。
- 默认界面字体和终端字体改为 `Consolas`，默认界面字号为 16。
- SFTP 标签打开时会创建后端 SFTP 会话，目录读取基于 `sftp_session_id`。
- SFTP 标签卸载时会关闭对应后端会话。
- SFTP 文件管理器支持刷新、双击进入文件夹、地址栏跳转、后退和前进。
- SFTP 路径输入支持 `~` / `~/` 跳转到远端 home 目录。
- SFTP 列目录时会显示加载状态和错误信息。
- SFTP 文件列表支持名称、大小和修改时间排序。
- SFTP 文件列表支持按设置切换大小显示方式：字节或自动单位。
- SFTP 文件列表显示修改时间，格式为 `yyyy-MM-dd HH:mm:ss`。
- SFTP 右键菜单支持刷新、下载、重命名、复制路径、删除、新建文件、新建文件夹、上传文件、上传文件夹。
- SFTP 新建、重命名、删除等基础写操作复用当前 `sftp_session_id`。
- SFTP 删除文件或目录前会弹出确认。
- SFTP 支持单文件上传、单文件下载、文件夹上传和文件夹下载。
- SFTP 传输任务会显示在固定高度队列中，并展示上传/下载状态、失败原因和传输进度。
- 上传服务器已有同名文件或文件夹时会先确认是否覆盖。
- SFTP 支持压缩远程文件或目录，支持解压 `.tar.gz` / `.tgz`。
- SFTP 支持打开和编辑小于 5MB 的文本类远程文件；编辑弹窗支持拖动、调整大小，并保存上次位置和尺寸。
- SFTP 支持把本地文件或文件夹拖到文件列表区域上传到当前目录。
- SFTP 传输队列支持取消进行中的上传或下载任务。
- 关闭 SFTP 标签时会先取消该标签仍在运行的传输任务，再关闭后端 SFTP 会话。
- SFTP 文件列表支持多选，并支持批量下载、删除、复制路径；批量压缩会把选中项写入同一个压缩包。
- 连接编辑器支持密码认证、私钥认证、私钥口令和分组字段。
- 连接编辑器的分组字段支持输入新分组，也支持从已有分组中选择。
- 私钥路径支持通过文件选择器填入。
- 连接编辑器支持 Redis 连接类型，可配置名称、分组、主机、端口、数据库编号和密码。
- Redis 连接支持保存到 `settings.json`，并通过 Rust 后端执行 `PING` 测试连接。
- Redis 连接在连接面板中显示为 `redis://host:port/db`，右键菜单可测试连接。
- 双击 Redis 连接会打开 Redis 工作区标签。
- Redis 工作区支持选择数据库编号、输入关键字模糊匹配并刷新扫描 key。
- Redis 工作区支持设置 key 分隔符，默认 `:`，已加载 key 会按分隔符展示为目录树。
- Redis 工作区显示当前 DB 的 key 总数和本次已加载数量；一次加载数量默认 5000，可在工具栏修改。
- Redis key 扫描由 Rust 后端通过 `DBSIZE` 获取总数，通过 `SCAN` 按加载上限扫描，并用 pipeline 批量读取类型和 TTL。
- Redis key 列表显示 key 名称、类型和 TTL。
- Redis key 支持双击打开只读详情弹窗，查看 `string`、`hash`、`list`、`set` 和 `zset`。
- Redis `string` 内容默认最多读取 5MB，集合类默认最多读取 500 条，避免大 key 阻塞界面。
- Redis 工作区支持新建 `string`、`hash`、`list`、`set` 和 `zset` 类型 key，新建时不会覆盖已有 key。
- Redis key 详情弹窗支持按 ESC 关闭。
- Redis `string` 类型支持在详情弹窗中编辑并保存内容。
- Redis key 详情支持设置 TTL、移除 TTL、重命名和删除 key，删除前会二次确认。
- Redis key 列表行右键菜单支持编辑、重命名和删除，删除前会二次确认。
- Redis key 重命名使用 Redis `RENAMENX`，目标 key 已存在时不会覆盖。
- 工作区标签栏支持横向滚动，标签标题完整显示且不压缩，滚动条隐藏。
- 标签栏空白区域保留底部分隔线，激活标签下方不显示分隔线。
- 已补充项目 `README.md`。
- 已补充 Apache License 2.0 的 `LICENSE` 文件。
- `package.json` 和 `src-tauri/Cargo.toml` 已声明 `Apache-2.0`。
- 已补充轻量本地 i18n，不引入额外 i18n 框架；主要工作台、设置、连接、终端和 SFTP 文案支持简体中文 / English。
